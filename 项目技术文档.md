# IdeaSpark AI 项目孵化平台 - 项目技术文档

## 一、项目概述

### 1.1 项目背景

在当前的"大众创业，万众创新"背景下，许多优秀的创意往往因为缺乏结构化的引导和专业文档的支持而胎死腹中。根据《2025年中国小微创业报告》显示，**83%的创业者在一年内失败**，68%的创业者面临方向定位不清晰的问题。

**IdeaSpark AI** 应运而生，旨在利用生成式 AI 技术，为创新团队提供一套**全流程、沉浸式、智能化**的项目孵化解决方案。

### 1.2 核心定位

- **使命**：降低从"灵感"到"立项"的门槛
- **定位**：AI 驱动的一站式项目工作站，集成了市场调研、产品设计、品牌塑造与商业分析
- **目标用户**：创新团队、独立创业者(OPC)、大学生创业团队

### 1.3 项目信息

| 项目属性 | 详情 |

| 项目名称 | IdeaSpark AI 项目孵化平台 |
| 版本号 | 0.0.0 |

### 3.2 核心功能详解

#### 3.2.1 AI 创意工坊 (AI Workshop)

AI 创意工坊是本项目的核心模块，建立了六大标准化孵化阶段，每个阶段都具备深度 AI 交互能力：

| 阶段 | 功能描述 | 输出产物 |

| **创意验证阶段** | 利用 AI 进行多维度痛点挖掘与用户画像建模 | 《需求分析说明书》 |
| **产品定义阶段** | 基于 MVP 原则进行功能裁剪，AI 自动生成 User Flow | 用户操作路径图 |

| **品牌叙事阶段** | AI 生成品牌故事与 Slogan，包括电梯演讲稿 | 品牌文案 |

#### 4.1.2 状态管理设计

采用 Pinia 进行模块化状态管理，核心 Store 包括：

**aiWorkshop Store** - AI 创意工坊状态管理：

```typescript
// 核心类型定义
export type ProjectModule = 'home' | 'idea' | 'product' | 'brand' | 'ui' | 'feasibility' | 'docs'

export interface ModuleData {
  key: ProjectModule
  label: string
  description: string
  checklist: StageChecklistItem[]
  artifacts: Artifact[]
  data?: Record<string, unknown>
}

// 核心状态
const projectList = ref<ProjectSummary[]>([])  // 项目列表
const currentProjectId = ref<string | null>(null)  // 当前项目ID
const currentModule = ref<ProjectModule>('home')  // 当前模块
const modules = ref<Record<ProjectModule, ModuleData>>({...})  // 模块数据
```

#### 4.1.3 Composables 设计

**useAiModuleData** - AI 模块数据响应式管理：

```typescript
export function useAiModuleData<T extends Record<string, unknown>>(
  moduleKey: ProjectModule,
  defaultData: T
): ToRefs<T> {
  const store = useAiWorkshopStore()
  const { modules } = storeToRefs(store)
  
  // 合并默认数据与已保存数据
  const savedData = modules.value[moduleKey].data || {}
  const formData = reactive({ ...defaultData, ...(savedData as Partial<T>) }) as T

  // 监听变化并同步到 Store
  watch(formData, (newValue) => {
    store.updateModuleData(moduleKey, { ...(newValue as Record<string, unknown>) })
  }, { deep: true })

  return toRefs(formData)
}
```

2. **空间归属模型**：将"个人空间"和"团队空间"统一抽象为 `teams` 表，通过 `is_personal` 字段区分

3. **插件化架构**：`plugins` 表作为元数据中心，`project_plugins` 表作为配置中心，实现能力与配置解耦

## 五、项目亮点

### 5.1 AI 原生 (AI-Native) 架构

项目放弃了传统的"侧边栏对话"模式，采用了**嵌入式 AI 引擎**：

- **上下文感知 (Context-Aware)**：AI 能够感知用户当前所处的业务上下文
- **精准建议**：点击表单旁的 AI 按钮即可获取针对当前输入项的精准建议
- **实时同步**：UI 表单数据与 AI 背景 Context 无缝同步

### 5.2 响应式多模态状态管理

利用 **Pinia** 深度定制了模块化状态机：

- 用户在 A 模块生成的"产品定位"，会自动作为 B 模块"视觉设计"和 C 模块"品牌文案"的 Prompt 预设参数
- 实现了全项目周期内逻辑的高度自洽

### 5.3 极具未来感的交互美学

- **视差滚动与动效**：深度应用 **GSAP** 与 **ScrollTrigger**，实现复杂的页面进入动画与交互反馈
- **全屏视频英雄区**：首页采用高性能 MP4 背景，营造出极强的视觉冲击力与"科技孵化"氛围

### 5.4 AI 测试技术落地

创新性地将 AI 应用于测试场景：

1. **逻辑自洽性交叉测试**：AI 自动读取各阶段数据，测试逻辑一致性，标注潜在逻辑断层
2. **商业模式压力测试**：模拟"极度保守"与"极度进取"两种投资者视角，测试商业计划在极端市场条件下的生存概率
3. **自动化合规性静态检查**：基于预置法规库，对品牌文案进行静态扫描，实时预警违禁词与合规风险
