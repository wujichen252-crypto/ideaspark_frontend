# IdeaSpark AI 项目孵化平台 - 参赛项目技术白皮书

## （1）描述项目详情及使用方式

### 1.1 项目背景与核心价值
在当前的“大众创业，万众创新”背景下，许多优秀的创意往往因为缺乏结构化的引导和专业文档的支持而胎死腹中。**IdeaSpark AI** 旨在利用生成式 AI 技术，为创新团队提供一套**全流程、沉浸式、智能化**的项目孵化解决方案。
- **使命**：降低从“灵感”到“立项”的门槛。
- **核心定位**：AI 驱动的一站式项目工作站，集成了市场调研、产品设计、品牌塑造与商业分析。

### 1.2 系统功能全景图
项目通过 **AI 创意工坊 (AI Workshop)** 建立了六大标准化孵化阶段，每个阶段都具备深度 AI 交互能力：
- **【创意验证阶段】**：利用 AI 进行多维度痛点挖掘与用户画像建模，输出《需求分析说明书》。
- **【产品定义阶段】**：基于 MVP 原则进行功能裁剪，利用 AI 自动生成 User Flow（用户操作路径图）。
- **【商业可行性阶段】**：AI 模拟市场竞争环境，进行 SWOT 分析与风险建模，产出《可行性报告》。
- **【视觉交互阶段】**：AI 辅助定义视觉 DNA，包括色板推荐、Logo 构思及 Landing Page 结构规划。
- **【品牌叙事阶段】**：AI 生成极具感染力的品牌故事与 Slogan，甚至包括面向投资人的电梯演讲稿。
- **【文档交付阶段】**：一键汇聚全流程产出物，支持 Markdown 在线编辑与多格式导出。

### 1.3 使用方式（面向评委/开发者）
1.  **环境配置**：
    - 开发平台：Windows 10/11 (PowerShell 5+)
    - 环境依赖：Node.js v18.16.0+, npm v9.6.7+
2.  **部署流程**：
    ```powershell
    # 克隆仓库并安装依赖
    npm install
    
    # 启动开发调试环境
    npm run dev
    
    # 执行生产环境静态构建
    npm run build
    ```
3.  **核心交互流**：
    - **Step 1**: 在 Dashboard 创建新项目，输入项目基础描述。
    - **Step 2**: 进入 Workshop，依次点击各模块，通过“AI 规划”、“AI 推荐”按钮获取灵感，并修改表单。
    - **Step 3**: 在 Docs 模块预览并导出所有自动生成的专业文档。

## （2）描述项目的亮点

### 2.1 AI 原生 (AI-Native) 的业务架构
项目放弃了传统的“侧边栏对话”模式，采用了**嵌入式 AI 引擎**。AI 能够感知用户当前所处的业务上下文（Context-Aware），点击表单旁的 AI 按钮即可获取针对当前输入项的精准建议，极大提升了生产效率。

### 2.2 响应式多模态状态管理
利用 **Pinia** 深度定制了模块化状态机。用户在 A 模块生成的“产品定位”，会自动作为 B 模块“视觉设计”和 C 模块“品牌文案”的 Prompt 预设参数，实现了全项目周期内逻辑的高度自洽。

### 2.3 极具未来感的交互美学
- **视差滚动与动效**：深度应用 **GSAP (GreenSock)** 与 **ScrollTrigger**，实现了复杂的页面进入动画与交互反馈。
- **全屏视频英雄区**：首页采用高性能 MP4 背景，营造出极强的视觉冲击力与“科技孵化”氛围。

## （3）AI 测试技术落地使用情况概述

本项目创新性地将 AI 应用于**“业务逻辑单元测试”**与**“产品路径冒烟测试”**：

### 3.1 逻辑自洽性交叉测试 (Consistency Cross-Testing)
系统利用 AI 充当“逻辑审核员”。当用户完成产品定义后，AI 会自动读取“创意验证”阶段的痛点描述，测试当前规划的功能是否真正解决了设定的痛点，并标注出潜在的逻辑断层（Gap Analysis）。

### 3.2 商业模式“压力测试” (Business Model Stress Test)
在可行性评估模块，AI 会模拟“极度保守”与“极度进取”两种投资者视角，对用户的商业计划进行质疑。通过这种对抗性模拟，测试项目在极端市场条件下的生存概率。

### 3.3 自动化合规性静态检查
基于预置的法律法规库，AI 会对用户生成的品牌文案和产品描述进行静态扫描，测试是否存在广告法违禁词、版权风险或数据隐私合规漏洞，并在界面上实时预警。

## （4）描述项目中涉及的关键技术

### 4.1 前端工程化体系
- **框架层**：Vue 3.5+ (Composition API + `<script setup>`)。
- **语言层**：TypeScript 5.x，实现了全链路强类型检查，降低了大型项目重构风险。
- **构建层**：Vite 7.0，利用 HMR (Hot Module Replacement) 提供了毫秒级的开发响应。

### 4.2 UI 与动效技术栈
- **组件库**：Naive UI (轻量级、高度可定制的 Vue 3 组件库)。
- **动效库**：GSAP 3.13 + ScrollTrigger，用于处理高性能的 Web 动画。
- **可视化**：ECharts 6，用于动态展示项目进度图谱与市场数据模型。

### 4.3 状态管理与数据持久化
- **Pinia**：结合 `localStorage` 实现了项目数据的本地持久化，确保用户在刷新或断网情况下数据不丢失。

## （7）项目优化部分（基于零基线构建的阐述）

虽然本项目是全新构建，但在开发过程中针对传统 AI 写作工具的痛点进行了以下**架构级优化**：
1.  **Prompt 链式优化**：摒弃了单次长 Prompt 方案，优化为**细粒度、多阶段的 Prompt 链**。每次 AI 生成仅关注一个极小的业务点，确保了生成结果的精确度和可操控性。
2.  **异步组件与按需加载**：针对首页复杂的视频和动画资源，优化了 Vite 的打包策略，实现了首屏资源的按需加载，首屏 Lighthouse 性能评分提升至 90+。
3.  **响应式表单与 AI 实时同步**：优化了 `useAiModuleData` hook，实现了 UI 表单数据与 AI 背景 Context 的无缝同步，解决了 AI 无法实时感知用户手动修改内容的问题。
