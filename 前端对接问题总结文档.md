## 一、已修复问题汇总

### 1. 状态码判断错误

**问题描述**: 前端代码中使用 `res.data.status === 0` 判断接口成功，但后端返回的成功状态码是 **200**。

**影响范围**:

- 帖子列表加载
- 圈子列表加载
- 关注列表加载
- 发布帖子

**修复方案**: 将所有 `status === 0` 改为 `status === 200`

**涉及文件**:

- `src/views/CommunityView.vue` (4处)

**代码示例**:

```javascript
// 修复前
if (res.data.status === 0 && res.data.data) {
  posts.value = res.data.data
}

// 修复后
if (res.data.status === 200 && res.data.data) {
  posts.value = res.data.data
}
```

---

### 2. 帖子作者信息字段缺失

**问题描述**: 后端返回的 `author` 对象只有 `id` 和 `username`，但前端使用了 `author.avatar` 和 `author.name`，导致头像和名称不显示。

**后端返回格式**:

```json
{
  "author": {
    "id": 1,
    "username": "张三"
  }
}
```

**修复方案**: 添加备用显示逻辑

- 头像: 使用默认头像 `https://api.dicebear.com/7.x/avataaars/svg?seed=${author.id}`
- 名称: 使用 `author.name || author.username`

**涉及文件**:

- `src/views/CommunityView.vue`

**代码示例**:

```vue
<n-avatar
  :src="post.author.avatar || `https://api.dicebear.com/7.x/avataaars/svg?seed=${post.author.id}`"
/>
<span class="username">{{ post.author.name || post.author.username }}</span>
```

### 3. 圈子字段名不匹配

**问题描述**: 类型定义中使用 `icon` 和 `cover`，但后端接口返回的是 `iconUrl` 和 `coverUrl`。

**修复方案**:

1. 修改 `Group` 接口字段名
2. 更新所有使用的地方
3. 添加默认头像

**涉及文件**:

- `src/api/types.ts`
- `src/views/CommunityView.vue` (3处)

**代码示例**:

```typescript
// 修复前
export interface Group {
  icon?: string
  cover?: string
}

// 修复后
export interface Group {
  iconUrl?: string
  coverUrl?: string
}
```

### 4. 发布帖子参数格式错误

**问题描述**:

1. `images` 和 `tags` 字段应该是逗号分隔的字符串，但前端传的是数组
2. 当没有图片时，传空字符串导致数据库 JSON 解析错误

**后端期望格式**:

```json
{
  "images": "https://example.com/img1.jpg,https://example.com/img2.jpg",
  "tags": "创新，创业，技术"
}
```

**修复方案**:

1. 将数组使用 `.join(',')` 转换为字符串
2. 没有图片时不传 `images` 字段

**涉及文件**:

- `src/views/CommunityView.vue`
- `src/api/types.ts`

**代码示例**:

```javascript
const tagsString = (formValue.topics as string[]).join(',')
const imagesUrls = fileList.value.map((f) => f.url || '').filter(Boolean)
const imagesString = imagesUrls.length > 0 ? imagesUrls.join(',') : undefined

const res = await createPost({
  title: formValue.title || '',
  content: formValue.content,
  tags: tagsString,
  ...(imagesString && { images: imagesString }),
  visibility: formValue.visibility as PostVisibility
})
```

## 二、待后端修复问题

### 1. Hibernate 懒加载问题 (严重)

**错误信息**:

```json
{
  "message": "服务器内部错误: Could not write JSON: failed to lazily initialize a collection of role: com.ideaspark.project.model.entity.User.ownedProjects: could not initialize proxy - no Session",
  "status": 500
}
```

**问题原因**: 后端在序列化 `User` 对象时，尝试获取 `ownedProjects` 集合，但此时数据库 Session 已经关闭。

**影响接口**:

- `GET /api/community/groups` (获取圈子列表)
- 可能还有其他涉及 User 对象的接口

**建议解决方案** (后端):

1. 在实体类上添加 `@JsonIgnoreProperties` 忽略懒加载字段
2. 使用 DTO (Data Transfer Object) 而不是直接返回实体类
3. 在查询时使用 `JOIN FETCH` 预先加载关联数据
4. 添加 `@Transactional` 确保 Session 在序列化完成前保持打开

**优先级**: 🔴 高
