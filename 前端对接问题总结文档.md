## 问题记录

### 问题 1：社区帖子列表接口 CORS 预检请求 502 错误

**发现时间**：2026-03-23

**接口信息**：

- 请求 URL：`http://47.108.232.238:9000/api/community/posts`
- 请求方法：OPTIONS (CORS 预检请求)
- 状态码：502 Bad Gateway
- 请求来源：`post.ts:28`

**问题描述**：
浏览器发送 CORS 预检请求 (OPTIONS) 时，Nginx 返回 502 Bad Gateway。这表明 Nginx 无法连接到后端服务。

**错误特征**：

- 预检请求 (preflight) 失败
- 响应大小：0.0 kB
- 耗时：约 5 秒（超时）

**可能原因**：

1. 后端服务未启动或已崩溃
2. Nginx 反向代理配置错误，无法连接到后端端口
3. 后端服务端口（9000）未监听
4. 防火墙或安全组阻止了连接

**解决状态**：❌ 待后端修复

**最新测试时间**：2026-03-23（问题仍然存在）

**建议**：

- 检查后端 Java 服务是否正在运行
- 检查后端服务是否监听 9000 端口
- 检查 Nginx 反向代理配置中的 upstream 地址是否正确
- 检查服务器防火墙是否允许 9000 端口访问

## 问题综合分析

### 关键发现

从浏览器网络面板可以看到：

| 接口      | 状态 | 类型      | 大小   | 耗时 |
| --------- | ---- | --------- | ------ | ---- |
| posts     | 502  | preflight | 0.0 kB | 5.0s |
| groups    | 502  | preflight | 0.0 kB | 5.0s |
| following | 502  | preflight | 0.0 kB | 5.0s |

**共性特征**：

1. **全部是 CORS 预检请求 (OPTIONS) 失败** - 不是实际的 GET/POST 请求
2. **全部是 502 Bad Gateway** - Nginx 无法连接到后端
3. **响应大小为 0** - 没有收到后端响应
4. **耗时约 5 秒** - 连接超时

### 根因判断

**后端服务完全不可用** - Nginx 反向代理配置正确，但无法连接到后端服务（端口 9000）。

可能情况：

1. 后端 Java 服务未启动
2. 后端服务启动但监听端口错误
3. 后端服务启动后崩溃
4. Nginx upstream 配置错误

### 排查步骤

```bash
# 1. 登录服务器，检查后端进程
ps aux | grep java

# 2. 检查端口监听情况
netstat -tlnp | grep 9000
ss -tlnp | grep 9000

# 3. 检查 Nginx 配置
cat /etc/nginx/conf.d/ideaspark.conf

# 4. 测试 Nginx 到后端的连通性
curl -v http://localhost:9000/api/health

# 5. 查看 Nginx 错误日志
tail -f /var/log/nginx/error.log

# 6. 查看后端应用日志
tail -f /var/log/ideaspark/application.log
```
