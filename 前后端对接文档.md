# IdeaSpark 前后端对接文档

## 项目概述

IdeaSpark 是一个创新项目管理平台，提供用户管理、团队协作、项目市场等功能。本文档详细描述了后端 API 接口，方便前端开发者进行对接。

## 技术栈

- **后端**：Spring Boot 3.4.1 + Java 17 + JPA + MySQL
- **认证**：JWT
- **文件存储**：阿里云 OSS
- **接口风格**：RESTful

## 基础信息

- **API 基础路径**：`http://47.108.232.238:9000/api`
- **认证方式**：Bearer Token（JWT）
- **响应格式**：统一 JSON 格式

### 统一响应格式

```json
{
  "status": 200,
  "message": "操作成功",
  "data": {}
}
```

| 字段    | 类型         | 说明        |
| ------- | ------------ | ----------- |
| status  | number       | HTTP 状态码 |
| message | string       | 响应消息    |
| data    | object/array | 响应数据    |

## 认证机制

所有需要认证的接口都需要在请求头中添加 `Authorization` 字段：

```
Authorization: Bearer {token}
```

## 详细接口文档

### 1. 用户管理

#### 1.1 用户登录

**接口**：`POST /api/user/login`

**请求参数**：

| 字段     | 类型   | 必填 | 描述     |
| -------- | ------ | ---- | -------- |
| email    | string | 是   | 用户邮箱 |
| password | string | 是   | 用户密码 |

**请求示例**：

```json
{
  "email": "user@example.com",
  "password": "password123"
}
```

**响应示例**：

```json
{
  "status": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",

      "username": "张三",

    }
```
```json
  }
}
```

## 错误处理

### 常见错误码

| 状态码 | 描述           | 示例消息             |
| ------ | -------------- | -------------------- |
| 400    | 请求参数错误   | "请求参数格式错误"   |
| 401    | 未授权         | "请先登录"           |
| 403    | 禁止访问       | "没有权限执行此操作" |
| 404    | 资源不存在     | "用户不存在"         |
| 500    | 服务器内部错误 | "服务器内部错误"     |

### 错误响应示例

```json
{
  "status": 401,
  "message": "请先登录",
  "data": null
}
```

## 开发环境配置

### 本地开发

1. 克隆项目
2. 配置 `application.yml` 中的数据库连接
3. 运行 `IdeaSparkApplication.java` 启动服务
4. 访问 `http://localhost:8080/ping` 验证服务是否启动

### 环境变量配置

在项目根目录创建 `.env` 文件：

```env
# 数据库配置
DB_URL=jdbc:mysql://localhost:3306/ideaspark?useSSL=false&serverTimezone=Asia/Shanghai
DB_USERNAME=root
DB_PASSWORD=your_password

# JWT 配置
JWT_SECRET=your_jwt_secret
```

## 部署说明

1. 打包项目：`mvn clean package -DskipTests`
2. 运行 JAR 文件：`java -jar target/ideaspark-0.0.1-SNAPSHOT.jar`
3. 或使用 Docker 部署

## 注意事项

1. **认证**：所有需要认证的接口都需要在请求头中添加 `Authorization: Bearer {token}`
2. **文件上传**：文件大小限制取决于服务器配置
3. **请求频率**：请合理控制 API 请求频率，避免过度请求
4. **数据安全**：敏感操作请确保用户身份验证和权限检查
