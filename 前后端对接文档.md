# IdeaSpark 社区模块接口文档

## 模块概述

社区模块提供帖子管理、评论管理、圈子管理和点赞功能，支持用户发布动态、互动交流、加入兴趣圈子等社交功能。

## 基础信息

- **基础路径**：`/api/community`
- **认证方式**：Bearer Token（JWT）
- **响应格式**：统一 JSON 格式

### 统一响应格式

```json
{
  "status": 200,
  "message": "操作成功",
  "data": {}
}
```

## 详细接口文档

### 1. 社区帖子

#### 1.1 创建帖子

**接口**：`POST /api/community/posts`

**描述**：发布新的社区帖子，支持关联项目、添加图片和标签

**认证**：需要

**请求参数**：

| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| title | string | 是 | 帖子标题 |
| content | string | 是 | 帖子内容 |
| images | string | 否 | 图片URL数组的JSON字符串，如 `["url1","url2"]` |
| tags | string | 否 | 标签数组的JSON字符串，如 `["tag1","tag2"]` |
| channel | string | 否 | 频道分类 |
| visibility | string | 否 | 可见性（PUBLIC/PRIVATE），默认PUBLIC |
| projectId | string | 否 | 关联项目ID |

**请求示例**：

```json
{
  "title": "分享我的新项目",
  "content": "这是一个很棒的项目...",
  "images": "[\"https://example.com/img1.png\",\"https://example.com/img2.png\"]",
  "tags": "[\"Vue3\",\"前端\"]",
  "channel": "项目分享",
  "visibility": "PUBLIC"
}
```

**响应示例**：

```json
{
  "status": 201,
  "message": "创建成功",
  "data": {
    "id": "post-id-1",
    "title": "分享我的新项目",
    "content": "这是一个很棒的项目...",
    "createdAt": "2024-01-01T00:00:00"
  }
}
```

---

#### 1.2 获取帖子列表

**接口**：`GET /api/community/posts`

**描述**：获取所有公开的社区帖子列表

**认证**：不需要

**响应示例**：

```json
{
  "status": 200,
  "message": "查询成功",
  "data": [
    {
      "id": "post-id-1",
      "title": "分享我的新项目",
      "content": "这是一个很棒的项目...",
      "images": [
        "https://example.com/img1.png",
        "https://example.com/img2.png"
      ],
      "tags": ["Vue3", "前端"],
      "channel": "项目分享",
      "visibility": "PUBLIC",
      "likesCount": 10,
      "commentsCount": 5,
      "viewsCount": 100,
      "createdAt": "2024-01-01T00:00:00",
      "updatedAt": "2024-01-01T12:00:00",
      "author": {
        "id": 1,
        "username": "张三",
        "avatar": "https://example.com/avatar.jpg"
      },
      "project": {
        "id": "project-id-1",
        "name": "创新项目"
      }
    }
  ]
}
```

**注意**：
- `images` 和 `tags` 返回的是数组格式，不是 JSON 字符串
- 后端会自动清理 URL 中的反引号、空格、换行符等
